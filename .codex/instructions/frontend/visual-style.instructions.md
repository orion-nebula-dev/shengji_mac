# Frontend Visual Style Instructions

## 视觉目标

界面应像 macOS 桌面生产力工具：克制、清晰、状态可信、信息密度合理。不要做营销式 landing page，也不要用装饰性视觉掩盖功能状态。

## 布局原则

1. 第一屏必须是可操作工作台。
2. 常用状态、待办、任务、转写、语义产物应可扫描。
3. 页面区域使用清晰的工具栏、列表、表格、侧栏、详情面板。
4. 避免卡片套卡片；卡片只用于重复项目、详情面板或工具容器。
5. 固定格式控件要有稳定宽高，避免 hover、加载、文字变化造成布局跳动。

## 控件原则

1. 二元设置使用 switch / checkbox。
2. 模式切换使用 tabs 或 segmented control。
3. 数值使用 input、stepper 或 slider。
4. 操作按钮可使用图标和短文本；工具按钮优先图标并提供 hover title。
5. 错误、加载、空态要可见，不用静默失败。

## 色彩与材质

1. 避免大面积单一色系，尤其是整页紫蓝渐变、深蓝灰、米色、棕橙。
2. 可以借鉴 Apple 当下液态玻璃 / macOS 风格，但要服务可读性。
3. 透明、模糊、阴影只用于层级，不用于装饰性堆叠。
4. 录音中、处理中、失败、成功等状态要有明确色彩语义。

## 文案

1. 默认中文。
2. 按钮文字短而具体，例如“开始录音”“处理待办”“测试连接”。
3. 错误提示说明阶段、原因和下一步。
4. 隐私边界要清楚表达：本地处理、云端语义、密钥保存、音频保留。
5. 不在界面中堆说明书式长文。

## 响应式

1. 当前 Tauri 主窗口最小宽度较大，但仍要避免固定超宽内容。
2. 文本不能溢出按钮、状态徽标、列表项。
3. 长路径、长模型名、长错误摘要要可换行或截断并保留 title。
4. 表格和列表要考虑空数据、加载中、失败、长内容。

## 视觉验证

涉及明显 UI 改动时：

1. 运行 `npm run build`。
2. 启动本地界面并用浏览器或 Tauri 预览检查首屏。
3. 检查至少默认态、加载态、失败态、长文本态。
4. 若用户要求设计参考图，可用 image generation 生成风格参考，但最终实现以实际界面截图验证为准。

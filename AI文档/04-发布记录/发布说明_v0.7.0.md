# 发布说明 v0.7.0

发布日期：2026-06-14

## 结论

v0.7.0 完成“待办中枢”版本范围：将 v0.6 的 `todo_extraction` 语义产物升级为可确认、可忽略、可追溯、可状态流转的正式 Todo 工作流。

## In Scope

1. 新增 `todo_candidates`，承接 `semantic_artifacts(type='todo_extraction')`。
2. 待办候选包含标题、详情、负责人、截止时间、优先级、置信度和来源片段。
3. 用户可接受候选进入正式 `todos`，也可忽略候选。
4. 正式 Todo 支持 `open`、`in_progress`、`done`、`dismissed` 状态。
5. 新增去重键，重复确认同一候选或相同标题不会静默生成多条正式任务。
6. 新增 Todo 来源追溯面板，展示来源片段和来源文稿。
7. 生成 Apple 风格视觉参考图并记录到版本归档。

## Out of Scope

1. 不实现 v0.8 思维脑图。
2. 不实现导出、同步和跨设备任务协作。
3. 不接入真实远端 M3 重新生成候选的编辑回写。
4. 不接入真实 Argmax / SpeakerKit 推理。
5. 不引入旧 Qwen / llama.cpp Todo runtime。

## 主要变更

1. 后端新增 `app::todo_service`，承载候选同步、接受、忽略、去重和状态流转。
2. `commands::todo` 收敛为薄命令层，新增候选同步、候选列表、接受候选、忽略候选和显式状态更新命令。
3. SQLite 新增 `todo_candidates`，并将 `todos.status` 迁移为 v0.7 状态集合。
4. 前端行动中心新增待确认候选区、状态筛选、负责人 / 截止 / 优先级、来源片段和来源文稿展示。
5. 浏览器原型 localStorage 会把旧 `pending/completed` 状态迁移为 `open/done`。
6. 版本号更新到 `0.7.0`。

## Code Review 结论

1. 行为回归：Todo 候选仍来源于 MiniMax M3 语义产物链路，未绕过 `semantic_artifacts`。
2. 状态管理：桌面端命令失败不乐观改写；浏览器原型仅用于预览 fallback。
3. 边界条件：旧 `pending/completed` Todo 会迁移为 `open/done`；重复确认同一候选返回已有正式 Todo。
4. 隐私边界：正式 Todo 和候选只记录来源片段 ID 与摘要文稿，不新增音频内容、API Key 或完整请求响应。
5. 可维护性：候选编排集中在 `todo_service`，命令层保持薄边界。
6. 剩余风险：DevTools 仍提示若干既有表单 label issue；不影响 v0.7 核心路径，但后续 UI 可统一补齐表单可访问性。

# 发布说明 v0.5.0

发布日期：2026-06-14

## 结论

v0.5.0 完成“转写与说话人评估”版本范围：新增本地音频导入评估入口、离线转写时间轴、说话人标签修正、错误片段标注、失败转写任务重试，以及本地模型缓存状态展示。

## In Scope

1. 建立 `transcript_segments` 时间轴字段、`speakers`、`speaker_segments`、`transcript_jobs`、`local_model_status` 表结构。
2. 新增 Rust transcript service 与 Tauri command 边界。
3. 新增 React “转写评估”页面，覆盖导入、时间跳转、说话人改名、错误标注、失败重试和本地模型状态。
4. 增加 v0.5 行为测试，覆盖本地音频导入、说话人修正、片段标注与失败任务重试。
5. 生成 Apple 风格视觉参考图并记录到版本归档。

## Out of Scope

1. 不下载或提交模型权重。
2. 不接入真实 Argmax local server / CLI 推理。
3. 不接入真实 SpeakerKit 声纹推理。
4. 不改变 MiniMax M3 作为 Todo 语义入口的边界。
5. 不引入旧 Qwen / llama.cpp Todo runtime。

## 主要变更

1. 后端新增 `app::transcript_service`，承载本地音频导入评估、说话人改名、转写片段标注和任务重试逻辑。
2. 后端新增 `commands::transcript`，暴露 `import_local_audio`、`get_transcript_review`、`rename_speaker`、`mark_transcript_segment`、`retry_transcript_job`。
3. SQLite 增加 v0.5 时间轴与说话人相关表，并为既有 `transcript_segments` 表补充轻量列迁移。
4. 前端新增 `转写评估` tab，复用现有 macOS 浅色玻璃风格，展示时间轴、说话人、模型缓存和任务状态。
5. 版本号更新到 `0.5.0`。

## Code Review 结论

1. 行为回归：新增 transcript command 未改变 Todo/MiniMax M3 语义链路。
2. 状态管理：前端桌面端优先调用 Tauri，浏览器模式使用默认评估样例，状态可本地交互闭环。
3. 边界条件：本地音频导入校验空路径与不存在路径；失败任务仅在失败且未超过重试上限时重排，达到最大重试次数时返回可见错误。
4. 可维护性：后端查询逻辑集中在 `transcript_service`，命令层只保留 payload 入口。
5. 剩余风险：真实 Argmax/SpeakerKit 推理仍未接入，当前为可替换的本地评估闭环。

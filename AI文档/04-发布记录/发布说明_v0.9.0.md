# 发布说明 v0.9.0 (2026.6.14)

## 结论

v0.9.0 交付“Moment 与深度研究”能力：系统可基于修正文稿、摘要和纪要生成关键 Moment 与 Deep Research 草稿，支持从转写片段发起研究，并将研究结论转为正式 Todo 或追加为脑图节点。

## In Scope

1. 新增 Moment：关键观点、决策点、分歧点、风险信号、高价值片段。
2. 新增 Deep Research 草稿：研究问题、背景、待验证假设、检索方向、可执行下一步。
3. 支持从某个转写片段发起研究。
4. 支持研究结论转为 Todo。
5. 支持研究结论追加为 mind map 节点，并生成新的 edited artifact。
6. 前端新增“价值发现”页面，覆盖 Moment、片段研究、草稿详情、来源追溯和转化动作。
7. 生成 Apple 风格视觉参考图，并保存到版本归档目录。

## Out of Scope

1. 不接入真实联网检索或外部资料抓取。
2. 不实现 v1.0 分享快照、Markdown / SRT / JSON 产品化导出。
3. 不实现 v1.1 多会话知识库、复盘周报或播客/TTS。
4. 不新增独立 `moments` 或 `research_drafts` 表；本版继续由 `semantic_artifacts` 承载。
5. 不重新引入旧 Qwen / llama.cpp Todo runtime。

## 主要变更

### 后端

1. `src-tauri/src/domain/artifact.rs`
   - 新增 `MomentDto`、`DeepResearchDraftDto`。
   - 新增 `StartResearchFromSegmentCommand`、`ConvertResearchToTodoCommand`、`AddResearchToMindMapCommand`。
   - `SemanticWorkbenchDto` 增加 `moments` 与 `deep_research`。

   - 新增 `generate_value_discovery`、`start_research_from_segment`、`convert_research_to_todo`、`add_research_to_mind_map`。
   - `moment` 和 `deep_research` 产物写入 `semantic_artifacts`，schema version 为 `v0.9`。
   - 研究转 Todo 使用研究 ID 去重，保留来源片段。
   - 研究加入脑图时生成新的 edited mind map artifact，不覆盖原脑图。
3. `src-tauri/src/commands/semantic.rs`
   - 新增 v0.9 Tauri command 薄层。
4. `src-tauri/src/lib.rs`

### 前端

   - 增加 Moment、Deep Research 和 v0.9 command 客户端类型。
2. `src/App.tsx`

   - 支持生成价值发现、从片段发起研究、研究转 Todo、研究加入脑图。

   - 新增 Moment 列表、研究草稿、来源追溯和转化动作样式。
4. `src/data/mock.ts`
   - 增加浏览器原型模式 v0.9 样例数据。

## Code Review

未发现阻塞问题。

已检查：

1. 版本范围：只实现 v0.9 Moment 与深度研究，不提前实现 v1.0 / v1.1。
2. 分层边界：command 保持薄层，业务逻辑在 `app/semantic_service.rs`，DTO 在 `domain/artifact.rs`。
3. 数据边界：复用 `semantic_artifacts(type='moment' | 'deep_research')`，未新增不必要表。
4. 转化行为：研究转 Todo 有去重键；研究加入脑图生成新 artifact，不覆盖用户脑图版本。
5. 前端状态：桌面端通过 `src/lib/desktop.ts` 调用 Tauri command，浏览器 fallback 仅用于原型预览。
6. 隐私：未新增 API Key、完整音频文件内容或外部资料抓取。
7. 旧路线：未重新引入 Qwen / llama.cpp Todo runtime。

Review 发现并处理：

1. v0.9 连续生成和片段研究会增加模型调用记录写入频率。已将 semantic service 的 model invocation ID 改为 session 内递增，降低同毫秒冲突风险。
2. 清理了上述调整后不再使用的旧 invocation ID helper，避免新增 dead code warning。

剩余风险：

1. Moment 和研究草稿当前仍是 deterministic 本地语义模拟，真实 MiniMax M3 结构化输出和 schema 校验留到后续 provider 增强。
2. 研究草稿暂由 `semantic_artifacts` 承载，后续若出现复杂研究版本树，可能需要独立表。
3. 来源片段按钮当前只跳转到转写评估页，尚未自动滚动定位到具体片段。

# 发布说明 v0.8.0 (2026.6.14)

## 结论

v0.8.0 交付“思维脑图”能力：基于 v0.6 修正文稿、摘要和纪要生成 `semantic_artifacts(type='mind_map')`，前端新增思维脑图工作台，支持节点编辑、折叠、来源追溯，以及 Markdown / JSON 导出。

## In Scope

1. 新增 mind map DTO：root、nodes、edges、summary、source_spans、edited、version、parent_artifact_id。
2. 新增后端 mind map service / command：
   - `generate_mind_map`
   - `update_mind_map_node`
   - `toggle_mind_map_node`
   - `export_mind_map`
3. 脑图产物写入 `semantic_artifacts(type='mind_map')`，schema version 为 `v0.8`。
4. 用户编辑节点或折叠节点时生成新的 edited artifact，不静默覆盖原始生成版本。
5. 前端新增“思维脑图”页面：
   - 脑图画布；
   - 边关系条；
   - 节点编辑器；
   - 来源片段入口；
   - Markdown / JSON 导出预览。
6. 生成 Apple 风格视觉参考图，并保存到版本归档目录。

## Out of Scope

1. 不接入真实在线 MiniMax M3 脑图生成调用；本版沿用本地 deterministic 语义服务模拟输出，保持 provider 边界。
2. 不新增独立 `mind_maps` 表，先由 `semantic_artifacts` 承载。
3. 不实现 Moment、Deep Research、分享快照、跨设备同步或多语言导出。
4. 不引入 React Router、Tailwind、Zustand 或大范围前端拆分。

## 主要变更

### 后端

1. `src-tauri/src/domain/artifact.rs`
   - 新增 `MindMapDto`、`MindMapNodeDto`、`MindMapEdgeDto`、`MindMapExportDto` 和节点更新命令 DTO。
   - `SemanticWorkbenchDto` 增加 `mind_map` 字段。
2. `src-tauri/src/app/semantic_service.rs`
   - 新增脑图生成、编辑、折叠、导出流程。
   - 生成和编辑均写入版本化 `mind_map` artifact。
   - Markdown / JSON 导出从 artifact payload 生成，不依赖前端拼接模型响应。
3. `src-tauri/src/commands/semantic.rs`
   - 新增对应 Tauri command 薄层。
4. `src-tauri/src/lib.rs`
   - 注册 v0.8 command。
   - 增加 v0.8 行为测试：生成、编辑不覆盖、重新生成不覆盖、Markdown / JSON 导出。

### 前端

1. `src/types.ts`、`src/lib/desktop.ts`
   - 增加 mind map 类型和 Tauri invoke 客户端。
2. `src/App.tsx`
   - 新增“思维脑图” tab。
   - 支持生成 / 重新生成、节点选择、节点编辑、折叠、来源片段跳转、导出预览。
3. `src/styles.css`
   - 新增脑图画布、节点、边关系、编辑器和导出预览样式。
4. `src/data/mock.ts`
   - 增加浏览器原型模式 v0.8 mind map 样例。

## Code Review

未发现阻塞问题。

已检查：

1. 版本范围：只实现 v0.8 思维脑图，不提前实现 v0.9 Moment / Deep Research。
2. 分层边界：command 保持薄层，业务逻辑在 `app/semantic_service.rs`，DTO 在 `domain/artifact.rs`。
3. 数据边界：复用 `semantic_artifacts(type='mind_map')`，未新增不必要表。
4. 编辑保护：节点编辑和折叠都会生成新 artifact，避免重新生成静默覆盖用户编辑。
5. 前端状态：桌面端通过 `src/lib/desktop.ts` 调用 Tauri command，浏览器 fallback 不掩盖桌面端失败。
6. 隐私：未新增 API Key、完整音频路径或完整用户转写日志输出。
7. 旧路线：未重新引入 Qwen / llama.cpp Todo runtime。

Review 发现并处理：

1. 功能 review 截图发现边关系条放在画布 grid 内时视觉像被裁切。已在第 1 轮聚焦修复中移到画布上方，并重新运行 `npm run build` 与功能 review 脚本。
2. Code review 发现后端读取最新脑图时不应吞掉 SQLite 错误，且节点标题不应只依赖前端校验。已在第 2 轮聚焦修复中改为显式 DB 错误返回，并增加后端空标题校验；修复后重跑 `cargo check`、v0.8 定向测试和 `npm run build`。

剩余风险：

1. 当前脑图生成仍是 deterministic 本地语义模拟，真实 MiniMax M3 mind map JSON 调用和 schema 校验留到后续 provider 增强。
2. 当前只在 artifact 内保存版本关系，未实现独立版本树或冲突合并。
3. 来源片段跳转目前进入转写评估页，尚未自动滚动定位到具体片段。

# 发布说明 v0.6.0

发布日期：2026-06-14

## 结论

v0.6.0 完成“转写修正与类型化纪要工作台”版本范围：新增修正文稿生成、原文 / 修正文稿对照、短语级修正记忆、录音类型与模板识别、摘要、纪要、待办候选、语义产物状态和模型调用记录展示。

## In Scope

1. 新增 v0.6 语义工作台后端编排：基于最新转写片段构建修正文稿、来源索引和结构化语义产物。
2. 写入 `semantic_artifacts`：`transcript_revision`、`recording_type`、`summary`、`meeting_minutes`、`todo_extraction`。
3. 写入 `model_invocations`，记录 MiniMax M3 语义边界的请求摘要、响应摘要和失败状态。
4. 新增短语级本地修正记忆，支持查看、启用 / 禁用、删除，不保存完整音频路径或完整转写稿。
5. 前端新增 `语义纪要` 页面，展示原文 / 修正文稿对照、拒绝某条修正、摘要、纪要、待办候选、语义产物和模型调用记录。
6. 生成 Apple 风格视觉参考图并记录到版本归档。

## Out of Scope

1. 不实现 v0.7 正式 Todo 接受、编辑、忽略和状态管理。
2. 不接入真实 Argmax local server / CLI 推理。
3. 不接入真实 SpeakerKit 声纹推理。
4. 不引入旧 Qwen / llama.cpp Todo runtime。
5. 不提交 API Key、生产配置、用户音频、SQLite 数据库或模型权重。

## 主要变更

1. 后端新增 `workspace_builder`、`transcript_revision_service`、`semantic_service`，承载 v0.6 语义工作台链路。
2. 后端新增 `commands::semantic`，暴露生成 / 读取工作台、修正记忆管理、语义产物重试和拒绝修正命令。
3. Domain 新增 `artifact` 与 `correction` DTO，前后端保持 camelCase 契约。
4. SQLite 新增 `transcript_correction_patterns` 表，用于短语级本地修正记忆。
5. 前端新增 `语义纪要` tab 和对应桌面 invoke 客户端，浏览器 preview 可使用 mock 数据完成功能 review。
6. 版本号更新到 `0.6.0`。

## Code Review 结论

1. 行为回归：未改变 MiniMax M3 作为 Todo 语义入口的边界，未引入旧 Qwen / llama.cpp。
2. 状态管理：前端桌面端优先调用 Tauri，浏览器 preview 使用 v0.6 mock 数据完成同路径演示。
3. 边界条件：M3 JSON 解析失败会写入 failed artifact，并提供重试入口；拒绝修正会更新 `transcript_revision` artifact 状态。
4. 隐私边界：修正记忆仅保存短语、替代表达、类型、scope 和置信度，不保存完整音频路径或完整转写文本。
5. 可维护性：后端编排集中在 `semantic_service`，命令层保持 payload 边界；DTO 与前端类型显式对齐。
6. 剩余风险：当前为离线可验证的 MiniMax M3 语义边界实现，真实远端 M3 解析质量、schema 强校验和成本观测仍需后续接入验证。

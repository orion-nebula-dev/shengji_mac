# 发布说明 v1.2.1 (2026.6.15)

## 结论

v1.2.1 完成 issue #11 的本地 ASR 与设置页优化：用户界面移除音频导入和云端 ASR 入口，设置页接入真实本地 ASR runtime 探测、模型下载状态管理和默认模型选择，录音片段成为转写查看入口。

## 主要变更

1. 设置页改为本地优先 ASR：探测 `argmax-cli` 与 `whisperkit-cli`，展示 runtime 状态、模型状态、缓存目录、下载进度和错误提示。
2. 默认模型为 `large-v3-v20240930_626MB`，并允许切换到 `base`、`tiny` 小模型。
3. 后端新增本地 ASR service / command / runtime infra，下载预热与转写优先委托 `argmax-cli`，不可用时回退到 `whisperkit-cli`。
4. SQLite 记录本地 ASR runtime 与模型状态，未验证模型不再默认显示为可离线使用。
5. CLI 模型预热流程成功后会校验本地模型目录；未找到模型文件时不会标记离线可用。
6. 转写重试前增加本地 ASR 就绪校验，未安装 runtime 或模型未下载时返回明确缺失提示。
7. 用户可见导入音频 UI 已移除，后端导入命令保留为开发与测试入口。
8. “转写评估”用户入口改为“录音片段”，录音片段和说话人列表提供最小可用查看能力。
9. 全局版本号统一显示在右下角，应用版本更新为 `1.2.1`。

```bash
git diff --check
```

已通过本地浏览器 smoke test：设置页能看到 `argmax-cli` / `whisperkit-cli`、默认模型 `large-v3-v20240930_626MB`、`base` / `tiny` 小模型；录音片段入口可见；导入音频、云端 ASR 和旧“转写评估”入口不可见；右下角仅显示 `v1.2.1`。

## 已知限制

1. 应用委托已安装的 `argmax-cli` / `whisperkit-cli` 执行模型预热下载和转写，不随安装包内置 CLI 或模型权重。
2. 当前版本实现下载链路、CLI 转写链路、缓存状态、runtime 探测和转写前置校验；复杂模型兼容矩阵和自动安装 CLI 仍为后续规划。
3. 录音片段与说话人列表为最小可用查看入口，编辑、合并、批量修正等高级操作后续迭代。
4. 浏览器原型模式只展示 mock 状态；真实探测、下载和转写需要在 Tauri 桌面端执行。

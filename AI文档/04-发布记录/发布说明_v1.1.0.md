# 发布说明 v1.1.0 (2026.6.14)

## 结论

v1.1.0 交付“翻译与多语言导出”能力：在不覆盖 v1.0 摘要和导出闭环的前提下，新增 `translation` 语义产物、转写翻译、摘要翻译、多语言导出模板和来源追溯。

## In Scope

1. 转写翻译：按修正文稿片段生成翻译结果，并保留 `sourceSegmentId`、说话人和时间范围。
2. 摘要翻译：生成独立摘要翻译，不覆盖原始 `summary` artifact。
3. 多语言导出模板：支持多语言 Markdown、JSON 和本地分享快照模板。
4. 翻译来源追溯：前端展示摘要来源和转写片段来源。
5. Export Center 增加 v1.1 翻译控制、目标语言选择和多语言导出入口。

## Out of Scope

1. 不实现播客脚本、TTS provider 或音频生成入口。
2. 不实现云端分享、外部同步或真实文件保存对话框。
3. 不引入旧 Qwen / llama.cpp Todo runtime。
4. 不改变 v1.0 导出中心的默认单语言导出闭环。

## 文档冲突处理

`AI文档/02-技术方案/声记-版本迭代与项目架构方案.md` 在 v1.1 方向中提到播客脚本和可选 TTS；`AI文档/03-版本迭代/声记-版本迭代目标与代码归档方案.md` 明确要求 v1.1 只做翻译与多语言导出，并排除 Podcast / TTS。按项目事实来源优先级，本版执行迭代计划，未创建播客脚本、TTS provider 或音频生成入口。

## 主要变更

1. 后端语义产物
   - `src-tauri/src/domain/artifact.rs` 新增翻译命令和 `TranslationArtifactDto`。
   - `src-tauri/src/app/semantic_service.rs` 新增 `generate_translation`，产物 schema 为 `v1.1`。
   - `src-tauri/src/commands/semantic.rs` 注册 `generate_translation` 命令。
   - MiniMax M3 provider 支持 `translation` artifact type。

2. 多语言导出
   - `src-tauri/src/domain/export.rs` 为导出命令增加 `target_languages`。
   - `src-tauri/src/app/export_service.rs` 在目标语言存在时生成多语言 Markdown、JSON 和快照模板。
   - 多语言导出内容引用 `translation` artifact，不读取完整本地音频路径或密钥。

3. 前端体验
   - `src/App.tsx` 在导出中心增加翻译与多语言导出控制区。
   - `src/types.ts`、`src/lib/desktop.ts` 和 `src/data/mock.ts` 增加翻译类型、Tauri client 和浏览器原型数据。
   - `src/styles.css` 增加翻译控制、来源追溯和响应式样式。

4. 版本与文档
   - 版本号更新为 `1.1.0`。
   - 新增 v1.1.0 发布说明、验收记录和归档清单。

## Code Review 结果

1. 行为回归：v1.0 默认导出路径保留；只有传入 `target_languages` 时才生成多语言模板。
2. 状态管理：前端翻译结果写入 `semanticWorkbench.translations`，并同步加入 artifacts 列表用于状态面板。
3. 边界条件：缺少翻译产物时多语言导出返回明确错误；空目标语言会回退到 v1.0 默认导出行为。
4. 隐私边界：导出快照和 JSON 不包含完整本地音频路径、API Key 或外部账号信息。
5. 范围边界：源码检查未新增 Podcast、TTS 或音频生成入口。
6. 剩余风险：当前翻译仍是 deterministic 本地语义模拟，真实 MiniMax M3 翻译质量、schema 强校验和成本观测留给后续 provider 增强。

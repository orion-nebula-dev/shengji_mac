## Code Review

1. 行为回归：Todo 语义入口仍固定为 MiniMax M3，未恢复旧 Qwen / llama.cpp。
2. 状态管理：语义工作台读写集中在 `SemanticWorkbench`，拒绝修正、重试 artifact、修正记忆变更均有可见反馈。
3. 边界条件：无语义会话时返回空工作台；失败 artifact 仅在 `failed` 状态允许重试；拒绝不存在 revision 返回错误。
4. 错误处理：命令层返回中文错误信息，前端通过 banner 呈现。
5. 可维护性：`semantic_service` 编排产物，`transcript_revision_service` 管修正文稿和修正记忆，`workspace_builder` 管最新转写上下文。
6. 性能风险：当前对照列表适合 v0.6 小规模样例，长音频后续需要分页或虚拟滚动。
7. 测试缺口：真实远端 M3 schema 兼容、token 成本、网络失败退避仍需后续版本补足。

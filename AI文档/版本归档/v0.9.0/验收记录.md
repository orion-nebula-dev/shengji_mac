# v0.9.0 验收记录

## 结论

v0.9.0 Moment 与深度研究版本已通过最小验证、code review 和功能 review。会话可生成 5 个 Moment，每个 Moment 包含时间戳和来源片段；Deep Research 草稿可从片段发起，并可转为 Todo 或追加为脑图节点。

## In Scope

1. `moment` / `deep_research` artifact。
2. 自动生成 3-10 个 Moment。
3. 每个 Moment 包含时间戳和来源片段。
4. Deep Research 草稿字段：研究问题、背景、待验证假设、检索方向、可执行下一步。
5. 从转写片段发起研究。
6. 研究结论转 Todo。
7. 研究结论加入脑图节点。
8. Apple 风格视觉参考图。

## Out of Scope

1. 真实联网检索。
2. 外部资料引用与报告生成。
3. v1.0 分享、导出产品化。
4. v1.1 多会话知识库、复盘周报、播客/TTS。
5. 独立研究表和复杂版本树。

## Code Review

未发现阻塞问题。

已检查项：

1. 版本目标与迭代计划一致，只实现 v0.9。
2. command / app / domain 分层没有倒置。
3. `semantic_artifacts` 与前端类型字段一致。
4. 研究转 Todo 保留来源片段并使用去重键。
5. 研究加入脑图生成新的 edited artifact，不覆盖既有脑图。
6. UI 覆盖默认态、生成态、片段发起、来源和转化状态。
7. 未新增敏感日志或生产配置改动。
8. 未重新引入旧 Qwen / llama.cpp Todo runtime。

已处理项：

1. semantic service 的 model invocation ID 已从时间戳改为 session 内递增，降低连续调用冲突风险。
2. 删除不再使用的旧 invocation ID helper，避免新增 dead code warning。

剩余风险：

1. Moment 与研究草稿当前是 deterministic 本地语义模拟。
2. 真实 MiniMax M3 研究草稿 JSON 解析、schema 校验和失败重试仍需后续增强。
3. 来源片段按钮未自动滚动定位到具体片段。

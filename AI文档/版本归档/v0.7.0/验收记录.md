手动路径：

1. 打开 `行动中心`。
2. 点击 `同步候选`。
3. 点击第一条候选 `接受`，候选显示 `已接受`，正式 Todo 列表出现对应行动项。
4. 点击第二条候选 `忽略`，候选显示 `已忽略`。
5. 选择正式 Todo，点击 `进行中`，状态持久化到详情面板。
6. 检查负责人、截止时间、优先级、来源片段和来源文稿可见。

备注：DevTools 提示既有表单 label issue；核心页面资源均为 200，不影响本版本用户路径。

## Code Review

1. 行为回归：未引入旧 Qwen / llama.cpp Todo runtime；正式 Todo 必须由候选确认进入。
2. 状态管理：候选、正式 Todo、详情面板和浏览器 fallback 状态保持一致。
3. 边界条件：旧状态迁移、重复确认、候选忽略和无候选空态均有处理。
4. 错误处理：桌面端命令失败会显示中文 banner，不静默乐观成功。
5. 性能风险：当前候选和 Todo 列表适合 v0.7 工作台规模；大量历史 Todo 后续需要分页或虚拟滚动。
6. 测试缺口：真实远端 M3 重新生成候选和复杂语义去重策略仍需后续验证。

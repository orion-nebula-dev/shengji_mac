## Code Review

1. 行为回归：未改变 MiniMax M3 Todo 语义入口，未引入旧 Qwen / llama.cpp。
2. 状态管理：前端浏览器模式与 Tauri 桌面模式均有明确路径，失败时回落为本地评估样例。
3. 边界条件：音频路径为空或不存在会返回错误；失败任务重试受 `retry_count < max_retry_count` 限制，达到最大次数时不会误报为已重新排队。
4. 错误处理：命令层返回中文错误信息，前端通过 banner 呈现。
5. 性能风险：时间轴当前为小样例列表，真实长音频后续需要虚拟滚动或分页。
6. 测试缺口：真实 Argmax/SpeakerKit 推理未接入，不能验证实际模型输出质量。

# v1.2.1 验收记录

## 结论

v1.2.1 已完成 issue #11 的核心验收：设置页与入口优化、本地 ASR runtime 探测、模型下载状态管理、转写前置校验、录音片段最小入口、用户可见导入音频 UI 移除和版本号更新。

## 验收项

1. 设置页展示本地优先 ASR，不再展示“火山云端 ASR”。
2. 设置页可见 `argmax-cli` 与 `whisperkit-cli` runtime 探测状态。
3. 默认模型为 `large-v3-v20240930_626MB`，并可切换到 `base`、`tiny`。
4. 应用记录本地 ASR 缓存目录、下载进度、离线可用状态、校验状态和错误提示。
5. CLI 模型预热下载流程成功后校验模型目录存在；未找到模型文件时不标记 `available`。
6. runtime 与模型就绪时，录音转写任务调用本地 CLI；未安装 runtime 或模型未就绪时，转写重试返回明确缺失提示，不再假设本地模型可用。
7. 用户可见“导入音频”入口已移除，开发/测试用后端导入命令保留。
8. “转写评估”入口改为“录音片段”，录音片段和说话人列表可查看基础数据。
9. 全局版本号仅在右下角显示 `v1.2.1`，代码版本号同步为 `1.2.1`。

## 已知限制

1. 本版本不内置 CLI、模型权重或自动安装器。
2. 下载/转写委托本机已有 `argmax-cli` / `whisperkit-cli`；下载链路使用官方 `transcribe --model ... --download-model-path ... --download-tokenizer-path ...` 预热参数，命令参数后续可随官方 CLI 变化继续收敛。
3. 浏览器原型模式不能执行真实 runtime 探测或模型下载，只展示 mock 状态。
4. 本次验收机器未安装 `argmax-cli` / `whisperkit-cli`，因此未执行真实模型下载；已通过官方 CLI 参数契约、fake runner 单元测试和缺失 runtime 检测覆盖下载链路行为。

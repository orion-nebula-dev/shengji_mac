# v0.4.0 验收记录

## In Scope

1. 后端按 v0.4 建立 commands / app / domain / infra / providers / jobs 分层地基。
2. ASR、Speaker、Semantic、Embedding、Export provider 边界可登记和独立配置。
3. Todo 入口固定为 MiniMax M3 语义边界，候选产物进入 `semantic_artifacts`。
4. 新增并迁移 `semantic_artifacts` 与 `model_invocations`。
5. 设置页展示本地 ASR / 本地说话人分离 / 云端 MiniMax M3 的隐私边界。
6. 删除旧 Qwen / llama.cpp Todo runtime 默认、兜底和 legacy 运行路径。

2. 不执行真实 SpeakerKit 说话人分离。
3. 不调用 MiniMax M3 生成语义产物。
4. 不进入 v0.5 音频导入、转写时间轴或 speaker label 评估页。
5. 不执行 `dev -> test -> main` 合版本、正式 tag 或 GitHub Release。

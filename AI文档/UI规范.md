# UI规范

## 1. 文档目标

本文档定义智能 Todo 一期的界面设计规范，目标是建立统一、轻量、可持续扩展的产品视觉与交互语言。整体风格要求为“小清新的工具风格”，强调清爽、克制、安静、易读。

## 2. 设计关键词

1. 小清新
2. 轻工具
3. 低打扰
4. 清晰分组
5. 柔和层次
6. 易读优先

## 3. 整体视觉方向

### 3.1 风格定义

界面应呈现“桌面效率工具”气质，而不是社交产品或重商务后台风格。视觉上应避免强烈攻击性颜色、过重阴影、复杂渐变和过度装饰。

### 3.2 视觉原则

1. 以浅色背景为主，整体明亮通透。
2. 使用低饱和度自然色系，避免高纯度荧光色。
3. 组件尽量轻边框、轻阴影、较大留白。
4. 信息密度中等偏低，优先保证扫描效率。
5. 交互反馈明显但不跳脱。

## 4. 色彩规范

### 4.1 主色建议

| 角色 | 建议色值 | 用途 |
| --- | --- | --- |
| 主色 | `#66C2A5` | 主按钮、选中态、关键状态强调 |
| 主色浅层 | `#E8F7F1` | 标签底色、选中背景、卡片浅高亮 |
| 辅助蓝 | `#7FB3D5` | 信息提示、次级强调 |
| 暖黄 | `#F6D98C` | 温和提醒、待处理轻提示 |
| 成功绿 | `#7BC47F` | 已完成状态 |
| 危险红 | `#E98B8B` | 删除、失败、错误提示 |

### 4.2 中性色建议

| 角色 | 建议色值 | 用途 |
| --- | --- | --- |
| 页面背景 | `#F7FBF9` | 应用主背景 |
| 卡片背景 | `#FFFFFF` | 面板、弹层、列表项 |
| 一级文字 | `#24333A` | 标题、核心信息 |
| 二级文字 | `#5F727A` | 说明文字、时间、备注 |
| 分割线 | `#DDE9E4` | 边框、分割线 |
| 禁用态 | `#B8C6C1` | 禁用文字、禁用边框 |

## 5. 字体与排版

### 5.1 字体建议

1. macOS 优先使用系统中文字体，如 `PingFang SC`。
2. 英文与数字可跟随系统字体。
3. 避免花哨字体和标题装饰字体。

### 5.2 字号层级

| 层级 | 建议字号 | 建议字重 | 用途 |
| --- | --- | --- | --- |
| 页面标题 | 24px | 600 | 页面主标题 |
| 区块标题 | 18px | 600 | 卡片标题、分区标题 |
| 列表标题 | 15px | 600 | Todo 标题、设置项标题 |
| 正文 | 14px | 400 | 常规内容 |
| 说明文字 | 12px | 400 | 备注、辅助说明、时间 |

### 5.3 排版原则

1. 标题与正文要形成明显层次。
2. 一屏内避免字号过多，建议不超过 4 种。
3. 文本左对齐优先，减少居中布局。

## 6. 布局规范

### 6.1 页面结构

一期建议采用双栏布局：

1. 左侧为导航与筛选区。
2. 中间为 Todo 列表主区域。
3. 右侧为详情与运行状态区。

设置页可采用卡片式纵向布局。

### 6.2 间距规范

| 项目 | 建议值 |
| --- | --- |
| 页面边距 | 24px |
| 模块间距 | 20px |
| 卡片内边距 | 16px |
| 表单项垂直间距 | 12px |
| 列表项间距 | 10px |
| 圆角 | 12px |

## 7. 组件规范

### 7.1 卡片

1. 使用白底卡片承载模块信息。
2. 卡片可带极轻阴影或浅边框。
3. 卡片标题区和内容区分层清楚。

### 7.2 按钮

1. 主按钮使用主色填充。
2. 次按钮使用白底描边。
3. 危险按钮仅用于删除、清空等高风险动作。
4. 不使用过强动效。

### 7.3 输入框

1. 输入框以浅边框和白底为主。
2. 聚焦态使用主色边框和柔和外发光。
3. 敏感字段如 `API Key` 默认掩码显示。

### 7.4 开关

1. 录音开关需视觉醒目。
2. 开启时使用主色或成功绿。
3. 关闭时使用中性色。

### 7.5 标签

1. `未完成` 使用浅绿色或浅黄色标签。
2. `已完成` 使用浅绿色灰化标签。
3. `失败` 使用浅红底标签。
4. `等待会话结束` 使用浅蓝色标签。

## 8. 页面级规范

### 8.1 设置页

设置页应分为以下卡片：

1. 录音设置卡片
2. 片段与触发规则卡片
3. 语音转写模型配置卡片
4. Todo 提取模型配置卡片
5. 系统状态与校验反馈卡片

### 8.2 Todo 列表页

Todo 列表页应包含：

1. 顶部状态栏
2. 筛选栏
3. Todo 列表区
4. Todo 详情区
5. 运行状态区

### 8.3 状态反馈

系统状态展示应尽量轻量，不应使用强干扰弹窗作为常态反馈。优先级如下：

1. 卡片内状态文字
2. 顶部轻提示条
3. 非阻断 Toast
4. 必要时再使用弹窗

## 9. 交互规范

### 9.1 反馈原则

1. 用户保存配置后应立即看到成功或失败反馈。
2. 录音开启后应持续展示“录音中”。
3. 会话处于等待触发阶段时，应展示“等待会话结束”。
4. Todo 新增后应以轻量高亮进入列表，而不是强动画弹出。

### 9.2 风险操作

1. 删除 Todo 需二次确认。
2. 清空缓存或清空历史记录需显式确认。
3. 修改模型配置后，如影响运行态，应提示“新配置将在下一轮任务生效”或即时生效规则。

## 10. 不推荐样式

1. 大面积高饱和蓝紫渐变。
2. 过厚阴影和玻璃拟态堆叠。
3. 强烈闪烁、跳动动画。
4. 过重后台管理系统风格。
5. 过于可爱化、卡通化的图标和插画。

## 11. 结论

智能 Todo 的 UI 应该给人“安静、清爽、可信赖”的感觉。它是陪伴型工具，不是内容平台。设计上要让用户长期开启也不觉得烦，重点是清晰状态、柔和视觉、低打扰反馈。

# v0.4 架构地基实施记录

文档状态：进行中
适用分支：`feature-v0.4.0-architecture-foundation`
首次记录日期：2026-06-13

## 1. 本 plan 目标

本 plan 只交付 v0.4 的架构地基，不接入真实 WhisperKit、SpeakerKit 或 MiniMax M3 生成链路。

已落地范围：

1. 新增 Rust provider catalog 与 provider trait 边界。
2. 新增独立 provider 配置字段：
   - `asr_provider_type`
   - `speaker_provider_type`
   - `semantic_provider_type`
   - `embedding_provider_type`
   - `export_provider_type`
3. 新增 MiniMax M3 语义配置字段：
   - `semantic_base_url`
   - `semantic_model_name`
   - `semantic_api_key_ref`
4. 新增统一 AI 产物与模型调用记录表：
   - `semantic_artifacts`
   - `model_invocations`
5. Todo 默认入口改为 `semantic_m3`，只登记 `semantic_artifacts(type='todo_extraction')` 边界，不再直接依赖旧本地 LLM。
6. 旧 Qwen / llama.cpp Todo 路径改为 `legacy_local_llm`，仅作为回滚路径保留。
7. 设置页展示本地 ASR、本地说话人分离、云端 MiniMax M3、Embedding 预留和本地导出的隐私边界。
8. 保存 image2 生成的 Apple 风格视觉参考图。

## 2. Provider 边界

| Provider | 能力 | 默认 ID | 隐私边界 |
| --- | --- | --- | --- |
| Local WhisperKit / Argmax | ASR | `local_whisperkit` | 音频默认留在本机，v0.5 接入实际 local server |
| Local SpeakerKit | Speaker | `local_speakerkit` | 本地生成 speaker label 与时间段，不识别真实姓名 |
| MiniMax M3 | Semantic | `minimax_m3` | 只发送转写后的文本上下文，用于摘要、Todo、脑图和研究 |
| Reserved Embedding | Embedding | `reserved` | v0.4 不启用默认路径 |
| Local File Export | Export | `local_file` | 导出默认写入用户选择的本地位置 |

Todo 相关说明：

1. `todo_provider_type` 默认值为 `semantic_m3`。
2. v0.4 处理 Todo job 时只登记 Todo 语义产物边界，实际 Todo 候选生成进入后续版本。
3. 旧本地 Todo 提取路径标记为 `legacy_local_llm`，不作为默认主线。

## 3. 新增表

### `semantic_artifacts`

用于统一承载摘要、Todo 候选、脑图、Moment、深度研究、翻译等 AI 产物。

关键字段：

1. `session_id`
2. `artifact_type`
3. `status`
4. `provider`
5. `model_name`
6. `schema_version`
7. `source_span_refs`
8. `payload_json`
9. `error_message`

### `model_invocations`

用于记录模型调用摘要，不记录完整隐私文本和完整密钥。

关键字段：

1. `provider`
2. `model_name`
3. `capability`
4. `status`
5. `request_summary`
6. `response_summary`
7. `input_tokens`
8. `output_tokens`
9. `duration_ms`
10. `estimated_cost_microunits`
11. `currency`
12. `error_message`
13. `trace_id`
14. `started_at`
15. `finished_at`

## 4. 非目标

本 plan 不做以下内容：

1. 不启动 Argmax / WhisperKit sidecar。
2. 不执行真实 speaker diarization。
3. 不调用 MiniMax M3。
4. 不生成摘要、脑图或深度研究产物。
5. 不完成 `src-tauri/src/lib.rs` 的全量物理拆分。

这些内容继续按 `AI文档/03-版本迭代/声记-版本迭代目标与代码归档方案.md` 进入后续 plan。

## 5. Plan 2：旧本地 LLM Provider 迁移

2026-06-13 完成第二个小步迁移：

1. 新增 `src-tauri/src/providers/semantic/legacy_local_llm.rs`，承载旧 Qwen / llama.cpp Todo 运行时逻辑。
2. `src-tauri/src/lib.rs` 不再保留本地 LLM 的 Prompt、manifest、JSON 解析、子进程请求等实现细节，只通过 `legacy_local_llm` provider 边界调用。
3. 单元测试改为直接验证 provider 边界：
   - 旧 `todo-embedded-v1` 版本归一化到 `qwen3-4b-instruct-2507-q4_k_m`
   - manifest 路径稳定
   - Prompt 注入、manifest 解析、运行时 Todo JSON 解析仍保持原行为
4. 默认 Todo 主线仍为 `semantic_m3`，`legacy_local_llm` 仅作为显式选择的旧路径保留。
5. Tauri 启动和 CLI job 入口不再无条件释放 legacy runtime 文件，只有设置为 `legacy_local_llm` 时才准备旧本地运行时资源。

## 6. Plan 3：主线 Provider 模块契约

2026-06-13 完成第三个小步迁移：

1. 新增 `src-tauri/src/providers/asr/local_whisperkit.rs`，定义 Local WhisperKit / Argmax 的本地 ASR provider descriptor、OpenAI Audio API 转写 path 和运行模式边界。
2. 新增 `src-tauri/src/providers/speaker/local_speakerkit.rs`，定义 Local SpeakerKit 的本地说话人分离 provider descriptor 和手动修正能力边界。
3. 新增 `src-tauri/src/providers/semantic/minimax_m3.rs`，定义 MiniMax M3 语义 provider descriptor、默认 base URL、默认模型名、上下文上限和支持的 artifact 类型。
4. `provider_catalog()` 改为复用各 provider 模块自己的 `descriptor()`，避免 catalog 与具体 provider 边界分裂。
5. `semantic_artifacts.artifact_type` 提前纳入 `transcript_revision`、`recording_type`、`meeting_minutes`，并为旧 CHECK 约束增加表重建迁移，确保 MiniMax M3 provider 支持列表与持久化 schema 一致。
6. 本 plan 仍不接真实 Argmax 进程、SpeakerKit 推理或 MiniMax M3 网络调用；真实能力进入后续 v0.5 / v0.6。

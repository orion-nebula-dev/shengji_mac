# 工作区目录规范

文档状态：v1.0
生成日期：2026-06-12
适用项目：`shengji_mac`

## 1. 结论

本工作区按“产品文档 + 前端应用 + Tauri 后端 + 本地资源 + 工程工具 + 版本归档”组织。代码版本通过 Git tag 归档，不在仓库内复制源码快照。

## 2. 根目录

```text
shengji_mac/
├── README.md
├── package.json
├── package-lock.json
├── index.html
├── vite.config.ts
├── tsconfig*.json
├── .gitignore
├── .github/
├── src/
├── src-tauri/
├── tools/
└── AI文档/
```

根目录只保留项目入口、构建配置和一级功能目录，不放临时文件、数据库、录音、模型权重和本地缓存。

## 3. 前端目录

```text
src/
├── main.tsx
├── App.tsx
├── types.ts
├── styles.css
├── views/
├── components/
├── lib/
└── data/
```

规则：

1. `App.tsx` 只负责顶层布局、导航和状态分发。
2. 页面级组件放入 `views/`。
3. 可复用 UI 放入 `components/`。
4. Tauri command 封装、格式化、artifact 适配放入 `lib/`。
5. mock 和开发样例数据放入 `data/`。

## 4. Tauri 后端目录

```text
src-tauri/src/
├── main.rs
├── lib.rs
├── commands/
├── app/
├── domain/
├── infra/
├── providers/
└── jobs/
```

规则：

1. `main.rs` 是 Tauri 启动入口。
2. `lib.rs` 只保留 app 初始化、状态注册和 command 注册。
3. `commands/` 只做参数校验、权限边界和 DTO 返回。
4. `app/` 放业务编排服务。
5. `domain/` 放核心实体、状态枚举和错误类型。
6. `infra/` 放 SQLite、HTTP、Keychain、文件、进程管理。
7. `providers/` 放 ASR、Speaker、Semantic、Export provider。
8. `jobs/` 放后台任务、队列和重试。

## 5. 文档目录

```text
AI文档/
├── README.md
├── 00-项目总览/
├── 01-产品方案/
├── 02-技术方案/
├── 03-版本迭代/
├── 04-发布记录/
├── 05-规范制度/
├── 06-验证报告/
├── 版本归档/
└── 废纸篓/
```

规则：

1. 新文档必须进入明确分类目录。
2. 发布说明统一放入 `04-发布记录/`。
3. 旧方案进入 `废纸篓/`，不直接删除。
4. 每个正式版本的验收记录进入 `版本归档/vX.Y.Z/`。

## 6. 工具目录

```text
tools/
├── minimax-capability-probe.mjs
└── minimax-capability-probe.test.mjs
```

规则：

1. 可重复运行的验证脚本放入 `tools/`。
2. 一次性临时脚本不进入仓库；如必须保留，需要改造成可重复运行脚本并补说明。
3. 工具脚本应有最小测试或明确验证命令。

## 7. 禁止入库

以下内容不进入 Git：

1. 本地录音文件。
2. SQLite 数据库和备份。
3. 模型权重和大二进制运行时。
4. `dist/`、`src-tauri/target/`、本地缓存。
5. API Key、完整用户转写文本、完整隐私路径。
6. `.DS_Store` 和本地分析工具缓存。

## 8. 版本归档

代码版本通过 Git tag 归档：

```bash
git tag -a vX.Y.Z -m "Release vX.Y.Z"
```

版本资料通过文档归档：

```text
AI文档/版本归档/vX.Y.Z/
├── 归档清单.md
├── 验收记录.md
└── 已知问题.md
```
